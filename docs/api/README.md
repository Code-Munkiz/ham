# HAM API — committed OpenAPI schema

This directory holds the **canonical OpenAPI schema** for the HAM FastAPI
backend (`src.api.server:fastapi_app`).

| File | What it is |
|---|---|
| `openapi.json` | OpenAPI 3.1 schema for every route mounted on the FastAPI app, generated by `scripts/export_openapi.py`. |

The runtime API still serves a live schema at `/openapi.json` and an
interactive UI at `/docs` when the server is running; the file in this
directory is a **static snapshot** so agents, code-gen tools, and
reviewers can read the API surface without spinning up the backend.

## Regenerating

The export script imports `src.api.server` in-process and calls
`FastAPI.openapi()`. It does **not** start a server, listen on a port,
or make outbound network calls. It pre-sets safe mock-mode environment
variables (mirroring `scripts/run_local_api.py`) so you do **not** need
production secrets, a live Hermes/OpenRouter gateway, Firestore
credentials, or Clerk JWT issuer configuration.

```bash
# From the repo root, with the project's virtualenv:
python scripts/export_openapi.py

# Verify the committed schema is in sync (CI-friendly; non-zero exit on drift):
python scripts/export_openapi.py --check
```

If the runtime API surface changes (new route, new request/response
model, version bump in `FastAPI(title="HAM API", version="…")`), run
the regenerate command and commit the updated `openapi.json` in the
same PR.

## What is **not** in this snapshot

- Auth-time behavior (Clerk JWT verification is disabled by the export
  script's mock defaults; route signatures are still complete).
- Provider-side schemas reached via the chat / Hermes gateway adapters —
  those upstream APIs have their own contracts (see
  `docs/HERMES_GATEWAY_CONTRACT.md`, `docs/FACTORY_DROID_CONTRACT.md`).
- Anything served by Vercel/Cloud Run infrastructure layers above the
  FastAPI app (CORS preflights, PNA middleware), which are documented
  in `src/api/server.py` itself.
