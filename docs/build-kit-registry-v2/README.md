# Build Registry v2 Docs

Start here:

- [Current State](CURRENT_STATE.md) — current UX, build-system, provider, and
  deployment posture.
- [Status & Handoff](STATUS.md) — detailed routed recipe status and validation
  commands.
- [Authoring Guide](AUTHORING_GUIDE.md) — rules for authoring registry
  recipes and modules.
- [HAM Legacy / Deprecation Audit](HAM_LEGACY_DEPRECATION_AUDIT.md) — first
  cleanup-pass audit and staged recommendations.

Many files in this directory are historical checkpoints, readiness reviews,
retrospectives, or outcome reports. Treat them as evidence for how the system
evolved, not as the current operator path unless `CURRENT_STATE.md` or
`STATUS.md` points to them directly.
